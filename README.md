# Kazakhstan Entrepreneurs Association — сайт (MVP)

Премиальный многоязычный MVP-сайт национальной ассоциации предпринимателей.
Чёрно-золотой дизайн, glassmorphism, плавные анимации, полностью на статике —
открывается без какой-либо установки или сборки.

## Что уже работает «из коробки»

- **Мультиязычность kk / ru / en**, отдельные URL `/kk/`, `/ru/`, `/en/`, переключатель
  языка на каждой странице, автоопределение языка браузера, hreflang, sitemap.xml.
- Все тексты выведены из `/locales/*.json` — ни одного текста не зашито в HTML напрямую.
- Главная страница целиком: hero, о нас, миссия и ценности, направления работы,
  статистика, 14 департаментов (карточки + модальные окна с услугами и формой),
  членство (3 пакета), региональная сеть, новости, мероприятия, партнёры,
  AI-ассистент (демо-чат), личный кабинет (демо-регистрация/вход), контакты.
- SEO: уникальные `<title>`/`description` на каждый язык, OpenGraph, hreflang,
  schema.org (Organization), sitemap.xml, robots.txt.
- Адаптивная вёрстка (мобильные/планшеты/десктоп), доступность (фокус, reduced-motion),
  контент всегда читаем даже без JavaScript (анимации — это лишь приятный бонус, а не
  условие видимости текста).

## Что является демо-режимом (и что нужно для реального запуска)

Серверной части и реальных интеграций нет — это статический фронтенд. Чтобы получить
полноценный продукт, на следующем этапе нужно:

| Раздел | Сейчас | Нужно для продакшена |
|---|---|---|
| Формы (консультации, контакты, департаменты) | Показывают "уведомление об успехе", никуда не отправляют данные | Подключить backend/API или сервис форм (например, отправка в CRM/почту) |
| AI-ассистент | Имитация ответа (одна и та же демо-фраза) | Подключить OpenAI API на сервере (никогда не хранить ключ в клиентском коде) |
| Личный кабинет | Вход/регистрация сохраняются в `localStorage` браузера, это не настоящая авторизация | Реальный backend с базой данных, хеширование паролей, сессии/JWT |
| Онлайн-оплата членства | Не реализована (только текстовые карточки пакетов) | Платёжный провайдер (Kaspi Pay, Stripe, локальный эквайринг) |
| Новости / мероприятия | Пустые блоки-заглушки ("раздел готов к подключению") | CMS или простая БД + админка для публикации материалов |
| CRM / ERP / Telegram / WhatsApp | Не подключены | Интеграция через серверные webhooks/API соответствующих сервисов |
| Логотип, фото, OG-обложка | Сгенерированы как простые временные плейсхолдеры в `/assets` | Заменить на реальные бренд-материалы |
| Домен | Все ссылки используют `https://site.kz` как placeholder | Заменить `BASE_URL` в `build.py` на реальный домен и пересобрать |

## Структура проекта

```
site/
├── build.py            # генератор статических страниц (читает locales/*.json)
├── locales/
│   ├── kk.json          # казахский (язык по умолчанию)
│   ├── ru.json          # русский
│   └── en.json          # английский
├── css/style.css        # премиальный стиль (чёрный/золото, glassmorphism)
├── js/main.js            # вся интерактивность (без зависимостей, чистый JS)
├── assets/               # логотип, favicon, og-обложка (временные плейсхолдеры)
├── kk/index.html         # сгенерировано build.py — не редактировать руками
├── ru/index.html         # сгенерировано build.py — не редактировать руками
├── en/index.html         # сгенерировано build.py — не редактировать руками
├── index.html             # редирект на язык браузера / сохранённый выбор
├── sitemap.xml
└── robots.txt
```

## Как добавить новый язык (Türkçe, 中文, العربية, Deutsch, Français, Кыргызча, Oʻzbekcha)

Архитектура изначально рассчитана на это:

1. Скопируйте `locales/en.json` в `locales/<код>.json` (например `locales/tr.json`)
   и переведите все значения (структуру и ключи не менять).
2. В `build.py` добавьте код языка в список `LANGS`.
3. Запустите `python3 build.py` — появится `/<код>/index.html`,
   автоматически обновятся hreflang, sitemap.xml и переключатель языка на всех страницах.

Никаких изменений в HTML/CSS/JS не требуется — это и есть смысл i18n-архитектуры.

## Деплой на Vercel + GitHub

В проекте уже лежит `vercel.json` — он настраивает чистые URL со слэшем на конце
(`/kk/`, `/ru/`, `/en/`) и долгое кэширование статики (`css/`, `js/`, `assets/`).
Никакой команды сборки на Vercel не требуется — это статический сайт.

Кратко: создайте репозиторий на GitHub → загрузите туда **содержимое** этой папки
(файлы `index.html`, `vercel.json`, папки `kk/`, `ru/`, `en/`, `css/`, `js/`,
`assets/`, `locales/`, `build.py`, `README.md` — каждый прямо в корень репозитория,
без обёртки) → на vercel.com нажмите Add New → Project → выберите этот репозиторий →
Deploy. Framework Preset — «Other», Build Command — пусто, Output Directory — `./`.

## Как изменить контент

Все тексты — в `locales/kk.json`, `locales/ru.json`, `locales/en.json`. Меняете
текст в JSON → запускаете `python3 build.py` → готово. Структуры (например список из
14 департаментов или 3 пакетов членства) — это массивы в том же JSON-файле, можно
добавлять/убирать элементы.

## Как пересобрать сайт после любых изменений

```bash
cd site
python3 build.py
```

## Как посмотреть сайт локально

```bash
cd site
python3 -m http.server 8080
# откройте http://localhost:8080/kk/  (или /ru/, /en/)
```

## Как опубликовать

Это обычная статика — подойдёт любой хостинг: Vercel, Netlify, Nginx, S3+CloudFront,
обычный shared-хостинг. Просто загрузите содержимое папки `site/` целиком (после
`python3 build.py`) в корень домена. Не забудьте поменять `BASE_URL` в `build.py`
на реальный домен перед финальной сборкой.

## Технологии

Чистый HTML/CSS/JS без зависимостей и без сборки (специально, чтобы сайт работал
сразу, без `npm install`). Если в будущем понадобится перейти на Next.js + next-intl
(как и было заложено в техническом задании) — JSON-файлы локализации в `/locales`
без изменений переносятся как messages-файлы next-intl, а секции из `build.py`
переносятся как React-компоненты. Архитектура была спроектирована так, чтобы
этот переход был прямолинейным.
